## Overview

<CardGrid stagger>
	<Card title="Generate Config Readme Files" icon="seti:markdown">
		Generate a readme file detailing configuration options to your dotnet application:
		```cs
		services
			.AddBindSight()
			.TransformationPipeline
			.UseDefaults()
			.Generators
			.AddReadme();
		```
	</Card>
	<Card title="Generate Json Schemata" icon="seti:json">
		Generate json schemata for configuration options to your dotnet application:
		```cs
		services
			.AddBindSight()
			.TransformationPipeline
			.UseDefaults()
			.Generators
			.AddJsonSchemata();
		```
	</Card>
	<Card title="Extensible Modular Architecture" icon="puzzle">
		Replace, extend, disable any behavior via DI, like the service providing the options to document:
		```cs
		services.AddSingleton<IBindSightGeneratorOptionsProvider, CustomOptionsProvider>()
		```
	</Card>
	<Card title="Built with Dependency Injection, for Dependency Injection" icon="star">
		All setup is done via DI, integrating natively into any dotnet application using the options pattern to bind configuration sources.
		The service collection targeted by our extensions will be analyzed for options pattern registrations that are bound against configuration sources.
	</Card>
	<Card title="Automatically Run Docs Generators at Application Startup" icon="rocket">
		Docs may be generated at any point in the application lifetime.
		A runner service may be resolved to run generators manually.
		A hosted service is provided to create documentation at application startup, optionally terminating the application after executing.
	</Card>
</CardGrid>
